# x402 — API de Pagamentos Programáticos

O Stellix implementa o padrão **HTTP 402 (Payment Required)** para monetização da sua API. Qualquer aplicação externa pode chamar endpoints do Stellix usando uma **API Key** pré-carregada com USDC. A cada chamada bem-sucedida, o custo correspondente é debitado automaticamente do saldo do usuário.

---

## Como funciona

```
Cliente → X-API-Key: sk_live_... → [x402Middleware] → rota protegida → resposta
                                          │
                                   verifica key hash
                                   checa saldo USDC
                                   executa rota
                                   debita custo (on finish)
                                   registra uso
```

### Fluxo detalhado

1. **Autenticação**: o cliente envia a chave via header `X-API-Key: sk_live_...` ou `Authorization: Bearer sk_live_...`.
2. **Validação**: o middleware calcula o SHA-256 da chave e busca no banco de dados. Chaves inativas retornam `403`.
3. **Verificação de saldo**: antes de executar a rota, o middleware consulta o saldo USDC do usuário. Se insuficiente, retorna `402 Payment Required`.
4. **Execução**: a rota é processada normalmente.
5. **Débito pós-resposta**: ao finalizar com status `2xx`, o custo é debitado do saldo principal do usuário. O uso também é registrado na tabela `api_usage` para auditoria.

---

## Custos por endpoint

| Método | Endpoint       | Custo (USDC) |
|--------|----------------|-------------|
| GET    | `/balance`     | 0.001       |
| GET    | `/statement`   | 0.001       |
| POST   | `/send`        | 0.01        |
| POST   | `/transfer`    | 0.01        |
| POST   | `/charge`      | 0.005       |

Endpoints sem custo definido (`ENDPOINT_COSTS`) têm custo `0` e não fazem débito.

---

## Endpoints disponíveis

Todas as rotas ficam sob o prefixo `/api/x402`.

### `GET /api/x402/balance`
Retorna o saldo USDC atual do usuário dono da API Key.

```json
{ "balance": 12.50, "currency": "USDC", "network": "testnet" }
```

### `GET /api/x402/statement`
Retorna as últimas 50 transações da conta.

### `POST /api/x402/send`
Envia USDC para um endereço Stellar externo.

```json
{ "recipient": "GXXXXX...", "amount": 5.0, "memo": "pagamento" }
```

### `POST /api/x402/transfer`
Transfere USDC entre contas internas do Stellix (por `userId`).

```json
{ "toUserId": 42, "amount": 2.0, "description": "split da conta" }
```

### `POST /api/x402/charge`
Cria uma cobrança com QR code e link de pagamento.

```json
{
  "title": "Assinatura Pro",
  "amount_usdc": 9.99,
  "description": "Plano mensal",
  "expires_in_hours": 24,
  "webhook_url": "https://meuapp.com/webhook"
}
```

Resposta inclui `qrData` (URL de pagamento) e `paymentAddress` (carteira Stellar de destino).

---

## Segurança

- **Chaves armazenadas como hash SHA-256** — o valor original nunca fica salvo em texto plano (o acesso ao valor completo exige descriptografia via AES-256-GCM e, opcionalmente, verificação 2FA).
- **Débito atômico pós-resposta** — o saldo só é debitado se a resposta for `2xx`, evitando cobranças por erros internos.
- **Auditoria completa** — toda chamada é registrada em `api_usage` com endpoint, método, custo, status e IP.

---

## Gerenciando API Keys

O Stellix expõe um painel de desenvolvedor (rota `/api/developer`) para:

- Criar e revogar API Keys
- Revelar a chave completa (com autenticação 2FA se habilitado)
- Ver logs de uso por chave (`/keys/:id/logs`)
- Consultar resumo de consumo (`/usage`, `/summary`)
- Configurar webhooks para eventos como `charge.paid` e `transfer.completed`

---

## Exemplo de uso com cURL

```bash
# Verificar saldo
curl https://stellixpay.com/api/x402/balance \
  -H "X-API-Key: sk_live_..."

# Criar uma cobrança
curl -X POST https://stellixpay.com/api/x402/charge \
  -H "X-API-Key: sk_live_..." \
  -H "Content-Type: application/json" \
  -d '{"title":"Pedido #123","amount_usdc":15.00}'

# Enviar USDC
curl -X POST https://stellixpay.com/api/x402/send \
  -H "X-API-Key: sk_live_..." \
  -H "Content-Type: application/json" \
  -d '{"recipient":"GXXXXX...","amount":5.00}'
```

---

## Resposta de erro 402

Quando o saldo é insuficiente, a API retorna:

```json
{
  "error": "Saldo USDC insuficiente",
  "balance": 0.003,
  "required": 0.01,
  "hint": "Deposite USDC na sua conta Stellix para usar a API"
}
```
